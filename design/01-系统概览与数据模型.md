# TDL 管理层待办系统 — 设计 Spec

## 一、系统定位

一个运行在云端的 AI 待办管理系统，通过飞书/钉钉机器人与管理层交互，覆盖**催收→处理→反馈**完整闭环。核心价值：让管理层的待办不被遗漏、不被拖延、被 AI 深度处理后产生行为洞察。

## 二、用户与阶段

| 阶段 | 用户范围 | 平台 | 周期 |
|------|---------|------|------|
| MVP | 少巍 1 人 | 飞书 | 2-4 周自我验证 |
| 推广期 | 10-30 人管理层 | 飞书 | MVP 跑通后逐步开放 |
| 成熟期 | 全员 + 跨平台 | 飞书+钉钉 | 推广期稳定后 |

## 三、TDL 数据模型

一条 TDL 待办的核心字段设计：

### 基础属性
| 字段 | 说明 | 示例 |
|------|------|------|
| 标题 | 任务简述 | "斯坦星球 Q3 课程大纲终审" |
| 描述 | 详细说明 | 富文本/语音转文字 |
| 负责人 | 飞书用户 ID | @shaowei |
| 参与人 | 协作成员列表 | @zhangsan, @lisi |
| 创建时间 | 自动生成 | 2026-05-14 09:30 |
| 截止时间 | 必填，触发催收 | 2026-05-20 18:00 |
| 来源 | 手动/飞书消息/审批/日历/文档 | manual / feishu_msg_xxx |

### 分类属性（广度+深度双维）
| 字段 | 说明 | 可选值 |
|------|------|------|
| 业务线 | 广度-业务域 | 励步英语 / 斯坦星球 / 飞书工资 / 诺伊曼 / 跨业务 |
| 职能域 | 广度-职能 | 市场 / 销售 / 人力 / 教务教学 / 行政 / 产品研发 |
| 流程阶段 | 深度-流程化程度 | 立项 / 执行中 / 验收 / 复盘 |
| 关键动作 | 深度-落地动作 | 可多选：审批/撰写/会议/调研/协调/审核 |
| 效果指标 | 深度-回收标准 | 文本描述，如"家长续费率提升 5%" |
| 收益预估 | 深度-收益预期 | 高/中/低 + 金额/影响描述 |
| 优先级 | 综合判定 | P0-紧急重要 / P1-重要不紧急 / P2-紧急不重要 / P3-常规 |

### 状态流转
```
新建 → AI已分析 → 待处理 → 进行中 → 已完成
                    ↓           ↓
                  已委派     已逾期 → 催收中 → 升级
```
