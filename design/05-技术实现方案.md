## 一、整体结构

系统跑在阿里云 ECS 上，通过飞书机器人与你交互。你的 Mac 不跑任何服务——合盖不影响。

```
飞书平台
  ├─ 机器人消息 ──→ 你的对话
  ├─ 日历 ────────→ TDL 自动写入日程
  └─ 消息推送 ←─── 提醒和卡片
        │
        ▼
阿里云 ECS (182.92.9.69)
  ├─ tdl-backend (FastAPI) ──→ Claude + ChatGPT
  ├─ PostgreSQL ──────────────→ TDL 数据
  ├─ APScheduler ─────────────→ 定时提醒
  └─ Nginx ──────────────────→ 反向代理
```

✅ 已确认：与励步钉钉 H5 项目部署在同一台 ECS，目录分离、服务分离、互不影响。

## 二、用什么技术，分别解决什么问题

### 必须有的

**① 云服务器（已有）**
- 阿里云 ECS 182.92.9.69，已在运行励步项目
- TDL 系统新增目录 `/opt/bots/tdl/backend`，新增 systemd 服务 `tdl-backend.service`
- 不需要额外买服务器

**② 后端框架：FastAPI**
- Python 异步框架，适合 AI API 调用密集的场景
- 自带数据校验和自动文档

**③ 数据库：PostgreSQL**
- 存 TDL 数据、管理层名单、审计日志、标签词典
- 开源免费，已有服务器上安装或复用

**④ 定时调度：APScheduler**
- 每日提醒、每周周报的定时任务
- 比 Celery 轻，MVP 不需要消息队列

**⑤ 飞书集成：官方 SDK**（生产）+ CLI（调试）
- 生产环境走飞书官方 Python SDK——稳定、错误处理好、支持高并发
- 调试运维时走飞书 CLI——快速验证、人类可读
- 直接复用 starplanet_ai 项目里的 `feishu_client.py`（250 行飞书 API 封装，与框架无关）

**⑥ 语音交互（MVP 核心，钉钉承载）**

平台选择：钉钉。理由——管理层主平台、你已有钉钉 H5 开发经验、Stream 模式天然适配 Realtime-2 WebSocket。

| 方式 | 说明 | 定位 |
|------|------|------|
| GPT-Realtime-2 实时对话 | 钉钉 H5 小程序 → WebRTC 捕获音频 → 后端 Relay 转 Realtime-2 WebSocket → AI 边听边拆 TDL | **主力**——"下周三前审核斯坦星球暑期班方案"直接说 |
| 钉钉语音消息兜底 | 发语音消息 → Whisper 转文字 → 进 AI 管道 | 降级——Realtime-2 不可用时无缝切换 |

💡 不做"先简单后复杂"的渐进——语音是 adoption 的前提，直接做最好的方案。详见 `参考-Realtime2语音交互平台对比.md`。

**⑦ AI 模型：OpenAI + DeepSeek 双模型**

| 角色 | 模型 | 提供商 | 说明 |
|------|------|--------|------|
| 主模型 | GPT-5 / GPT-4o | OpenAI | 会议纪要拆解、TDL 分类、周报生成 |
| 备选模型 | DeepSeek V3 Pro | DeepSeek | 中文场景补充、国内网络兜底 |

**fallback 策略**：
- 关键任务（会议纪要拆解、标签分类）：主模型失败提示"稍后重试"，不自动降级
- 非关键任务（简单摘要、提醒文案）：允许 fallback 到备选模型
- 单模型超时 30 秒，API Key 存 `.env`

💡 MVP 不做五模型池——模型越多测试矩阵越复杂，关键链路不可预测。

### MVP 不需要的

| 东西 | 干什么的 | 什么时候加 |
|------|---------|-----------|
| Web 可视化仪表盘 | 网页看趋势图 | 不做 Web。用飞书多维表格做仪表盘——和薪资系统同平台，不需要额外登录 |
| 备用模型（Hermes/OpenClaw） | 主模型挂了兜底 | 五模型池已覆盖，不需要额外备用 |
| 钉钉 H5 实时语音（Realtime-2） | 说话就能创建 TDL | V1.1，MVP 跑通后第一个迭代 |
| — | — | — |
| 向量数据库 | AI 长期记忆 | 第 2 阶段，数据量上来后再加 |
| OKR 对齐 | AI 关联战略目标 | 需要有真实维护的 OKR 数据源才行 |

## 三、从现有项目能复用哪些

| 资产 | 来源 | 复用方式 |
|------|------|---------|
| `feishu_client.py` | starplanet_ai 项目 | 直接复制——纯 HTTP 调用，跟框架无关 |

| ECS 部署模式 | 励步项目 | systemd + Nginx + git pull 部署，完全复用 |
| SSH Key | 已有 `id_ed25519` | 直接复用 |

## 四、密钥管理

⚠️ 真实密钥放 `.env` 文件（不提交 Git），业务参数放 `config/*.yaml`（可以提交 Git，不含敏感信息）。

```
项目根目录/
├── .env                    ← 不进 Git：API Key、数据库密码、飞书 App Secret
├── config/
│   ├── tdl_rules.yaml      ← 进 Git：置信度阈值、自动创建条件
│   ├── tag_dictionary.yaml ← 进 Git：标签词典
│   ├── escalation_policy.yaml ← 进 Git：升级策略参数
│   └── feishu_config.yaml  ← 进 Git：卡片样式、消息模板
```

## 五、AI 安全护栏

| 护栏 | 做法 |
|------|------|
| 不编造信息 | 标签词典做约束，AI 只能从已有的业务线/人员里选 |
| 高风险需确认 | 升级通知、批量提醒、修改他人任务 → 人工确认后才执行 |
| 成本控制 | 每人每日 token 上限，超了暂停 AI 调用 |
| 审计日志 | 每次创建、修改、催收、升级都留痕，可追溯 |
