## 一、一条 TDL 长什么样

### 1.1 核心字段（创建时系统必须有，大部分可由 AI 自动填充）

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 任务标题 | "斯坦星球暑期班课程方案审核" |
| `owner_id` | 负责人（飞书/钉钉 user_id） | @shaowei |
| `due_at` | 截止日期。如果是时间段则用 `due_start` + `due_end` | 单日：2026-05-20；范围：05-18 ~ 05-20 |
| `participants` | 参与协作的其他成员 | @zhangsan, @lisi |
| `status` | 当前状态（中文） | 见 §三：草稿 / 进行中 / 已逾期 / 已暂缓 / 已升级 / 已完成 / 已取消 |
| `priority` | 优先级 | P0-紧急重要 / P1-重要不紧急 / P2-紧急不重要 / P3-常规 |

💡 title 不需要模板——你正常说话就行。AI 从自然语言里提取。「下周三前把斯坦星球暑期班方案审完」→ AI 提取为"斯坦星球暑期班课程方案审核"。

### 1.2 AI 自动填充的字段

| 字段 | 说明 | 可选值来源 |
|------|------|-----------|
| `business_line` | 业务线 | 见 §二-2.1 |
| `function_domain` | 职能域 | 见 §二-2.2 |
| `product_line` | 产品线 | 见 §二-2.3，可选 |
| `stage` | 流程阶段 | 立项 / 执行中 / 验收 / 复盘 |
| `key_actions` | 关键动作 | 见 §二-2.4，可多选 |
| `tags` | 横向标签 | 见 §二-2.5，可多选 |
| `confidence` | AI 分类置信度 | 0.0 ~ 1.0，< 0.7 标"待确认" |

### 1.3 协作与依赖

| 字段 | 说明 |
|------|------|
| `waiting_for` | 等谁的反馈（user_id） |
| `blocked_by` | 被哪条 TDL 阻塞（引用另一条 TDL） |
| `parent_id` | 父任务（大任务拆子任务时用） |
| `completion_criteria` | 完成标准——做到什么程度算完成 |

💡 `waiting_for` 和 `blocked_by` 不同：前者是被动等（等教研组回意见），后者是硬阻塞（大纲没批不能写教案）。

### 1.4 结果回收（人工 + AI 共建，可选）

| 字段 | 说明 |
|------|------|
| `outcome_kpi` | 效果指标，如"家长续费率提升 5%" |
| `roi_estimate` | 收益预估：高 / 中 / 低 |

这部分人先填个大概，AI 帮你校准——比如你说"效果不错"，AI 追问"不错具体指续费率提升多少？"

### 1.5 系统内部字段

人不看、AI 和后台用：`tdl_id`（唯一编号）、`start_at`、`snooze_until`（暂缓恢复时间）、`source`（manual / dingtalk_msg）、`calendar_event_id`（系统自动在日历上创建的事件 ID）、`created_by`、`created_at`、`updated_at`、`audit_log`

## 二、标签词典

AI 分类 TDL 时只能从以下值里选。选不了的标"待确认"，不瞎猜。

|------|------|

**斯坦星球**：STEM（2.5-6岁）、CODE（6-10岁）、PythonAI（9-15岁）、C++双师（10-16岁）

✅ 已确认：VEX IQ 有业务文档，VEX GO 正在推进招生。IDEALAB 目前未投入运营，暂不纳入产品线标签。

### 2.5 横向标签

悬挂在所有分类之上：

| 标签 | 含义 |
|------|------|
| `@waiting_for` | 等某人反馈 |
| `@blocked` | 有硬阻塞 |

| `@delegated` | 已委派给他人 |

## 三、一条 TDL 的一生（状态流转）

```
草稿 ──→ 进行中 ──→ 已完成
  │          │
  │          ├──→ 已逾期 ──→ 已暂缓 ──→ 进行中
  │          │       │
  │          │       └──→ 已升级
  │          │
  │          └──→ 已取消
  │
  └──→ 已取消（草稿 24 小时未确认自动取消）
```

| 状态 | 含义 |
|------|------|
| 草稿 | AI 生成，等确认 |
| 进行中 | 正在推进 |
| 已逾期 | 过了截止时间，系统自动标记 |
| 已暂缓 | 用户主动延迟，到时恢复 |
| 已升级 | 已通知上级 |
| 已完成 | 搞定 |
| 已取消 | 不做了 |
