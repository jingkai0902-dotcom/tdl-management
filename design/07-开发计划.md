# 07 · 开发计划

> 与 06 的关系：06 定义了 MVP 范围，07 把它拆成可执行的开发批次。
> 与 05 的关系：05 的技术选型（FastAPI + PostgreSQL + APScheduler）决定了这里的文件路径和模块职责。

---

## 技术骨架一览

| 层 | 选型 |
|----|------|
| 后端框架 | FastAPI (Python 3.11+) |
| 数据库 | PostgreSQL 15 + SQLAlchemy 2.0 async + Alembic |
| 调度器 | APScheduler |
| 飞书 SDK | 复用 starplanet_ai 的 `feishu_client.py` |
| AI 客户端 | 新建 `ai_client.py`（Claude + ChatGPT 双模型） |
| 配置 | `.env` 存密钥 / `config/*.yaml` 存业务参数 |
| 部署 | systemd + Nginx + git pull（与励步项目同模式） |

## 批次 1：项目骨架 + 数据库 + 配置 + 审计日志

**依赖**：无（第一批）

### 做什么
- FastAPI 最小入口 + 健康检查端点
- PostgreSQL 建表：TDL 主表、AuditLog 表、预留沉淀字段
- Alembic 迁移配置
- 四个 YAML 配置文件 + `.env` 加载

### 验证方法
- `uvicorn app.main:app` 启动成功
- `GET /health` 返回 `{"status":"ok","db":"connected"}`
- `alembic upgrade head` 建表成功
- 手动插一条 TDL → 查出来字段完整
- 插一条审计日志 → 关联到 TDL

## 批次 2：飞书机器人收发 + 卡片交互

**依赖**：批次 1

### 做什么
- 飞书事件回调：接收消息、验签、路由
- 飞书卡片构建：确认卡片、已创建卡片、追问卡片
- TDL CRUD：创建草稿、确认创建、自动创建、查询
- 复用 starplanet_ai 的 `feishu_client.py`

### 验证方法
- 飞书给机器人发一段测试消息 → 收到回复卡片
- 手动创建一条 TDL → 数据库出现记录 + 审计日志
- 点卡片按钮 → 状态正常流转
- 飞书验签通过，伪造请求被拒绝

## 批次 3：AI 处理管道 + 会议纪要拆解（MVP 核心）

**依赖**：批次 2

**优先级**：最高。月会决议拆解是权重最高的入口。

### 做什么
- `ai_client.py`：Claude + ChatGPT 双模型客户端，带 fallback
- `ai_service.py`：
  - `intake_extract`：从自然语言提取结构化字段
  - `classify_tags`：对照标签词典分类
  - `detect_missing`：检测缺字段
  - `suggest_next`：下一步动作
  - `parse_meeting_minutes`：**会议纪要专属——输入长篇纪要文本，拆出决议清单→任务→责任人→截止时间→验收标准，批量生成 TDL 草稿**
- 标签词典做 AI 输出约束
- 三护栏：输出护栏（不编造）+ 高风险护栏（必须确认）+ 成本护栏（token 上限）

### 验证方法
- 输入「下周三前审核斯坦星球暑期班方案」→ 准确提取字段
- **输入励步5月月会纪要文本 → 批量产出 ≥5 条 TDL 草稿，每条含负责人+截止时间+业务线**
- 编造不存在的业务线 → AI 不输出不在标签词典中的值
- Claude 挂了 → 自动 fallback 到 ChatGPT

## 批次 4：催收调度 + 逾期追问 + 暂缓

**依赖**：批次 2（不等批次 3 全部完成，催收逻辑不依赖 AI）

### 做什么
- APScheduler：每日 08:30 触发催收，每周一 09:00 触发周报
- 催收逻辑：查询到期/逾期 TDL → 按升级策略路由
- 追问卡片：已完成/延期/需协助 按钮
- 暂缓：snooze_until 字段支持

### 验证方法
- 创建明天到期的 TDL → 明天 08:30 收到飞书提醒
- 逾期 2 天 → 收到追问卡片，点「已完成」→ status 变 done
- 逾期 3 天未回复 → 收到升级通知
- 说「明天再提醒」→ snooze_until = 明天 09:00，不再提醒，明天恢复
- 重启系统 → 调度器自动恢复

## 批次 5：TDL 自动写入日历

**依赖**：批次 2

### 做什么
- TDL 创建（自动或确认）后，调日历 API 在负责人日历上生成日程事件
- 日程内容：TDL 标题 + 截止时间 + 链接回 TDL
- 记录 `calendar_event_id`，方便后续取消/更新 TDL 时同步日历

### 验证方法
- 创建一条 TDL 并确认 → 负责人日历上出现对应日程事件
- 取消 TDL → 日历事件同步取消
- 两人协作 TDL → 两人日历上都有该事件

---

## 批次 6：周报事实账本

**依赖**：批次 1（有数据就能跑）

### 做什么
- 统计查询：新增/完成/逾期/延期/等待/阻塞/超 3 天未推进/下周到期
- 按业务线分布 + 按等待对象分组
- 每周一 09:00 飞书卡片推送

### 验证方法
- 数据库有本周 15 条 TDL（12 完成 3 逾期 2 等待）→ 周报数字准确
- 超 3 天未推进的 TDL 正确列出
- 周一 09:00 自动收到周报卡片

---

## 批次 7：部署到 ECS + systemd + Nginx + 验收

**依赖**：批次 1-6 核心闭环跑通

### 做什么
- systemd 服务文件 `tdl-backend.service`
- Nginx 反向代理配置（域名 `tdl.bdteach.cn` 或现有域名子路径）
- 一键部署脚本
- Smoke test

### 验证方法
- `systemctl status tdl-backend` → active (running)
- `curl http://127.0.0.1:8000/health` → ok
- 飞书给机器人发「测试」→ 收到回复
- 创建 TDL → 数据库有记录
- `journalctl -u tdl-backend` 无 ERROR
- 励步后端 `libu-backend.service` 不受影响
- 重启服务器 → tdl-backend 自动启动
