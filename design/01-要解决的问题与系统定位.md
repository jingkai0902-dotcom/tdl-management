## 一、核心问题的深层本质

### 表层问题

少巍管理两条公司业务（励步英语、斯坦星球）和一条自研技术线（飞书工资系统等 AI 开发项目）。每条线下有教务教学、销售、市场、人力、行政等多个职能域。日常待办来自：月度薪资数据汇总与发放、每周教学磨课、新人过课与授课水平确认、课后总结信息图与备课指南 PDF 的生产、AI 项目的持续开发。

管理层约 15 人，分布在投资人、校区校长、教务主管、教学主管、人力经理、销售/市场经理等角色上。

### 深层问题

好的公司，要为组织打造系统的结构能力。一个好的管理系统在于：**流程清晰、管理有节奏、决策可落地、落地能回收、回收有效果**。

以前这些靠具体的人来把控。但实际业务中，由于各人主动性差异、能力水平差异、注意带宽差异，很多管理动作掣肘，效能大打折扣。

TDL 作为工具，本可以统筹管理动作的流转和质量，潜移默化提升个人管理水平。但问题在于：它还是会落到个人的差异性上——让 TDL 本身的执行落地难以持续和有质量，经常流于形式，徒增工作负担。

### AI 解决的六个实质难点

1. **深度思考**：大模型的推理能力，让任务分析不再依赖人的经验和注意力
2. **可控标准**：Skill 和 Agent 机制，让管理动作变得可标准化、可复制
3. **自动化推进**：提醒、催办、回收、升级——替代了靠人盯、靠意志力的环节
4. **反馈复盘**：AI 融合管理工具方法论，指导管理层能力提升
5. **自动运转**：企业车轮可以自动化跑起来，有沉淀，可迭代
6. **交互效率**：语音等自然交互降低录入门槛

## 三、平台策略（待进一步验证）

平台不是预先分配功能，而是根据实际能力匹配。当前认知：

### 钉钉
- 管理层日常办公主平台
- 有日历、审批、机器人、H5 小程序
- 已有成功案例：之前做过销售人员上传通话录音 + AI 分析的应用
- 语音交互落地可能性待验证

### 飞书
- CLI 能力强大，自动化功能好
- 多维表格是目前薪资系统的载体
- 适合做 TDL 的数据管理、协作、AI 复盘、数字资产沉淀
- 日历功能可用
- 语音交互目前不太好落地

### 判断原则

不预设哪个平台做什么。核心问题是：TDL 的分配和落地需要什么能力，哪个平台能提供这些能力。

### 语音交互

气泡交互的语音转文字可做过渡方案。OpenAI Realtime-2 如果能量产应用，效果会更好。放哪个平台——看哪个能落地。

- 钉钉机器人开发的权限你有吗？之前做过钉钉应用，应该已经开通了？
- 飞书的多维表格分析——你设想 TDL 数据怎么跟多维表格结合？

## 五、理论基础

系统设计以 **GTD（Getting Things Done）** 五步流程为框架：

```
Capture → Clarify → Organize → Engage → Reflect
（捕获）   （厘清）   （组织）   （执行）  （反思）
```

落在 TDL 系统里，映射为五个模块：

| GTD 步骤 | TDL 系统模块 | 一句话 |
|----------|------------|--------|
| Capture 捕获 | 意图捕获 | 把脑子里的事倒出来 |
| Clarify + Organize 厘清+组织 | 结构化处理 | AI 帮你理清楚优先级、节奏、依赖 |
| Engage 执行 | 进度推进 | 自动提醒、追问、升级，替代人盯 |
| Reflect 反思 | 复盘可见 | 每周一看：做了什么、卡在哪里 |
| —（组织层面延伸） | 沉淀进化 | MVP 后上线：组织经验、效能、数字资产 |

## 七、系统持续进化原则

这套系统不是一个一次性开发项目。它应该：
- 持续关注社区（GitHub 优秀开源项目）、官方（飞书/钉钉 API 更新）、行业（管理方法论演进）
- 功能按阶段交付，但架构预留扩展空间
- 所有建议和方案不只是基于个人认知——需要参考成熟方案和社区最佳实践
