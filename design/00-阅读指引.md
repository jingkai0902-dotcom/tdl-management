# 00 · 阅读指引

## 这些文档是什么

TDL 管理层待办系统的完整设计方案。7 篇文档，从"为什么做"到"怎么写代码"，逐层展开。

## 文档地图

```
00-阅读指引          ← 你在这里
    │
01-要解决的问题      为什么做、给谁用、分几个阶段
    │
02-数据模型与标签    系统里记录什么信息、怎么分类
    │
03-功能模块          五个模块分别做什么、怎么协作
    │
04-交互方式          日常怎么跟系统对话
    │
05-技术实现方案      用什么技术、部署在哪里
    │
06-MVP实施方案      第一版做什么、不做什么
    │
07-开发计划          分几个批次开发、每批验收什么
```

## 怎么看

- **快速了解全貌**：读 00 → 01 → 06（知道 MVP 范围）就够了
- **细读每个设计点**：按顺序 01 → 02 → 03 → 04 → 05 → 06 → 07
- **只看一个具体问题**：跳到对应文档，每篇开头有「这篇跟其他文档的关系」

## 每篇文档会标注

| 标注 | 含义 |
|------|------|
| ⚠️ 待你确认 | 我从知识库里推断的，需要你验证或修正 |
| ✅ 已确认 | 你明确说过的事实或决定 |
| 💡 设计考量 | 为什么这么设计而不是那样，不确认也没关系 |
