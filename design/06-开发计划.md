# 06 · 开发计划

> 与 03 的关系：03 定义了五个功能模块和 MVP 范围，06 把它拆成可执行的开发批次。
> 与 05 的关系：05 的技术选型决定了这里的文件路径和模块职责。

---

## 技术骨架一览

| 层 | 选型 |
|----|------|
| 后端框架 | FastAPI (Python 3.11+) |
| 数据库 | PostgreSQL 15 + SQLAlchemy 2.0 async + Alembic |
| 调度器 | APScheduler |
| 钉钉 SDK | 钉钉 Stream 模式（机器人消息）+ H5 小程序（语音） |
| 飞书 SDK | 飞书日历 API（日历写入）+ 多维表格（仪表盘） |
| AI 客户端 | `ai_client.py`（五模型池：Claude → ChatGPT → DeepSeek → 智谱 → 通义千问，带运行时自动切换） |
| 配置 | `.env` 存密钥 / `config/*.yaml` 存业务参数 |
| 部署 | systemd + Nginx + git pull（与励步项目同模式，路径 `/opt/bots/tdl/backend`） |

## 批次 1：项目骨架 + 数据库 + 配置 + 审计日志

**依赖**：无

### 做什么
- FastAPI 入口 + `/health` 端点
- PostgreSQL 建表：TDL 主表（含 `linked_goal`、`calendar_event_id`、沉淀预留字段）、AuditLog 表
- Alembic 迁移
- 五个 YAML 配置 + `.env` 加载（含五模型 API Key）

### 验证
- `uvicorn app.main:app` 启动成功
- `GET /health` → `{"status":"ok","db":"connected"}`
- `alembic upgrade head` 建表成功
- 手动插一条 TDL → 字段完整

## 批次 2：钉钉机器人收发 + 卡片交互

**依赖**：批次 1

### 做什么
- 钉钉 Stream 模式事件回调：接收消息、验签、路由
- 钉钉卡片构建：确认卡片、已创建卡片、追问卡片、上级简报卡片
- TDL CRUD：创建草稿、确认创建、自动创建、查询
- 搭配飞书日历写入（TDL 创建后自动生成日历事件）

### 验证
- 钉钉给机器人发消息 → 收到卡片回复
- 手动创建 TDL → 数据库有记录 + 审计日志 + 日历事件
- 点卡片按钮 → 状态正常流转

---

## 批次 3：AI 处理管道 + 会议纪要拆解 ★ MVP 核心

**依赖**：批次 2

### 做什么
- `ai_client.py`：五模型池，运行时自动切换，故障冷却 5 分钟
- `ai_service.py`：
  - `intake_extract`：字段提取
  - `classify_tags`：对照标签词典分类
  - `detect_missing`：缺字段追问
  - `suggest_next`：下一步建议
  - `parse_meeting_minutes`：会议纪要 → 决议清单 → 批量 TDL 草稿
- 护栏：输出约束（不编造）+ 高风险确认 + 成本上限

### 验证
- 输入「下周三前审核斯坦星球暑期班方案」→ 准确提取字段
- 输入励步 5 月月会纪要文本 → 批量产出 ≥5 条 TDL 草稿，含负责人+截止时间
- 编造不存在的业务线 → AI 不输出
- 主模型挂了 → 自动 fallback

---

## 批次 4：催收调度 + 逾期追问 + 暂缓

**依赖**：批次 2

### 做什么
- APScheduler：每日 08:30 触发（按班制路由时间），每周一 09:00 周报
- 催收逻辑：到期/逾期查询 → 按升级策略路由
- 追问卡片：已完成/延期/需协助
- 暂缓：snooze_until 支持
- 每日完成确认：提醒里包含昨日完成数

### 验证
- 明天到期的 TDL → 明天 08:30 收到钉钉提醒
- 逾期 2 天 → 追问卡片，点「已完成」→ status 变 done
- 逾期 3 天未回复 → 升级通知
- 说「明天再提醒」→ snooze_until 生效

## 批次 5：TDL 自动写入日历

**依赖**：批次 2

### 做什么
- TDL 创建后 → 调日历 API → 负责人日历上生成日程
- 日程含标题 + 截止时间 + TDL 回链
- 取消/更新 TDL → 同步日历

### 验证
- 创建 TDL → 日历有事件
- 取消 TDL → 日历事件消失
- 协作 TDL → 多人日历均有

---

## 批次 6：周报事实账本

**依赖**：批次 1

### 做什么
- 统计查询：新增/完成/逾期/延期/等待/阻塞/超 3 天未推进/下周到期
- 按业务线分布 + 按等待对象分组
- 末尾推送一条「💡 下周建议」

### 验证
- 数据库有本周 15 条 TDL → 周报数字准确
- 超 3 天未推进的 TDL 正确列出
- 周一 09:00 自动推送

---

## 批次 7：钉钉 Realtime-2 语音交互 ★ MVP 核心

**依赖**：批次 2 + 3

### 做什么
- 钉钉 H5 小程序：WebRTC 捕获音频 → 后端 Relay → Realtime-2 WebSocket
- 后端 Relay：转发音频流 + 回调 function calling 创建 TDL
- 兜底：钉钉语音消息 → Whisper 转文字 → 进 AI 管道

### 验证
- 钉钉 H5 说话 → 实时对话中 AI 确认字段 → TDL 草稿生成
- Realtime-2 不可用 → 语音消息 → TDL 正常生成

## 批次 8：部署

**依赖**：批次 1-7 核心跑通

### 做什么
- `tdl-backend.service` + Nginx + deploy.sh
- Smoke test

### 验证
- `systemctl status tdl-backend` → active
- `curl /health` → ok
- 钉钉机器人发消息 → 有回复
- 励步 `libu-backend.service` 不受影响
- 重启服务器 → tdl-backend 自启
