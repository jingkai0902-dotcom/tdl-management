# 03 · 功能模块

> 与 02 的关系：每个模块操作的数据结构定义在 02。
> 与 04 的关系：每个模块的用户交互方式定义在 04。

---

## 概述

基于 GTD 五步流程校准后的五模块架构：

```
记下来 ──→ 理清楚 ──→ 推得动 ──→ 看得见 ──→ 长出来
 意图捕获    结构化处理   进度推进    复盘可见    沉淀进化
   ✅          ✅          ✅         ✅         🔮
  MVP         MVP         MVP        MVP       第2阶段
```

---

## 模块一：意图捕获（记下来）

### 三个输入通道

**① 会议纪要转 TDL**（✅ MVP 核心）——权重最高的入口。

月会后用户贴入会议纪要文本，AI 自动拆出：决议（decision）→ 任务（tdl）→ 责任人 → 截止时间 → 验收标准，批量生成 TDL 草稿。一次月会产出几十条有战略价值的 TDL。不做实时会议转写——只做纪要文本的后处理。数据结构按 meeting → decision → tdl 三层分开。

**② 日常对话录入**（✅ MVP 轻量）——管理层全员可用。给钉钉机器人发一句话生成个人 TDL。极轻版本：纯文本，只建个人 TDL。不做复杂查询、不做委派、不做语音。目的是验证执行层是否有人愿意天天用。

**③ TDL 自动写入日历**（✅ MVP）——TDL 创建后，系统在负责人日历上生成日程事件。打开日历就能看到待办。实现方式是**个人授权后代写**：每位管理者首次点一次授权，之后系统才能把 TDL 写入其个人钉钉日历。

### 低风险自动创建，高风险草稿确认

```
自动创建（发「已创建，可修改」卡片）：
  条件全部满足：截止时间明确 + 负责人明确 + 置信度≥85% + 不涉及他人

草稿确认（发「确认创建」卡片）：
  任一满足：缺截止时间、缺负责人、置信度低、涉及他人
```

## 模块二：结构化处理（理清楚）

### AI 做四件事

**① 字段抽取**：从自然语言提取 title、due_at、owner、completion_criteria

**② 标签分类**：对照标签词典（02 文档 §三）打标签，置信度 < 70% 标「待确认」不硬塞

**③ 会议纪要专属**：`parse_meeting_minutes`——输入长篇纪要文本，拆出决议清单 → 任务 → 责任人 → 截止时间 → 验收标准，批量生成 TDL 草稿（关联 meeting_id 和 decision_id）

**④ 缺失检测与下一步动作**：缺截止时间追问，缺完成标准只在必要时问

## 模块三：进度推进（推得动）

### 提醒节奏（按班制路由）

系统根据每个人的班制类型决定提醒时间：

| 班制 | 每日提醒 | 周二特殊 |
|------|---------|---------|
| 教师班 | 08:30 | 正常 |
| 运营班 | 08:30 | 延至 10:00（周二仅 4.5h 工时） |
| 标准班 | 08:30 | 正常 |
| 上一休一 | 有班日 08:30 | 休息日不推 |

每日提醒含昨日完成确认：「昨天完成了 3 条 ✅」

### 升级（MVP 轻量）

```
逾期 1 天 → 提醒本人
逾期 2 天 → 追问本人，带按钮：「[✅ 已完成] [📅 延期] [🆘 需协助]」
逾期 3 天 → 标记"待关注"，进入周报
手动升级 → 任何人可手动将 TDL 升级给上级或李珍
```

推广期再加：自动职级链升级、协作链通知、上级简报卡片。

### 升级语气

系统不说「逾期」，说「需要支持」：

| 场景 | 系统的说法 |
|------|-----------|
| 普通逾期 | "X 任务可能需要关注" |
| 升级通知 | "X 任务可能遇到了困难，已同步给 @上级" |
| 阻塞任务 | "X 任务建议协同推进" |

### 暂缓

用户说「明天再提醒」→ `snooze_until` = 明天 09:00，到时恢复。

## 模块四：复盘可见（看得见）

### 周报（每周一 09:00 钉钉卡片推送）

只做事实统计，不做行为分析：

```
📊 TDL 周报（5.11-5.17）
├─ 本周新增：15 条
├─ 本周完成：12 条（80%）
├─ 逾期未清：3 条
├─ 延期处理：2 条
├─ 等待他人：4 条（@张三 2条，@李四 1条…）
├─ 阻塞中：1 条
├─ 超过 3 天未推进：
│   ├─ 励步招生方案终稿（5 天未动）
│   └─ 斯坦星球课程大纲修订（4 天未动）
├─ 下周到期：6 条
├─ 卡点提示：飞书工资相关 TDL 连续 2 周有阻塞
└─ 💡 下周建议：飞书工资线连续 2 周阻塞，建议下周单独约谈
```

### MVP 不做这些（第 2 阶段以后）

- 高效时段分析、精力曲线、拖延模式识别
- 团队排名
- Web 可视化仪表盘——用飞书多维表格代替
- 老板督导航面板

---

## 模块五：沉淀进化（长出来）—— 第 2 阶段

模块五在 MVP 不开发功能，但数据库从第一天就预留字段：实际耗时、延期次数、关联业务成果。数据积累够了，第 2 阶段直接计算，不需要补数据。

---

## MVP 不做清单

- 钉钉 H5 实时语音（Realtime-2）—— V1.1
- Web 可视化仪表盘——用飞书多维表格代替
- 长期行为记忆与画像——数据量不够时是噪音
- OKR 自动对齐检查——没有真实维护的 OKR 数据源
- 审批流/文档深度嵌入——平台工程，非 MVP 核心
- 团队公开排名——推广后才考虑
- 复杂日历联动——仅做单向写入（TDL→日历），不做双向同步

🔮 第 2 阶段的五项（已记录不遗忘）：
- 老板督导航面板（本周必盯 N 件事 + 管理者风险名单）
- 管理者能力成长建档（沉淀进化模块的数据维度之一）
- TDL 模板结构校准 AI（三层目标：业务→管理→成长）
- 周期性项目与会前准备能力（详见 07-暂存需求决策记录）
