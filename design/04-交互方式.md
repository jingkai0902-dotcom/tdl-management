# 04 · 交互方式

> 与 03 的关系：03 定义了每个功能模块做什么，04 定义用户怎么跟这些功能交互。
> 与 03 的关系：MVP 范围定义在 03 末尾的「不做清单」，交互方式基于钉钉机器人卡片。

---

## 一、交互原则

**越轻越好。** 所有操作通过钉钉机器人的对话和卡片完成，不需要装新 App、不打开网页。语音交互（Realtime-2）在 V1.1 加入。

---

## 二、六种交互场景

| 你做什么 | 怎么触发 | 系统怎么回应 |
|---------|---------|------------|
| 创建 TDL | 钉钉给机器人发消息 | 高置信度→返回「已创建」卡片；低置信度→返回确认卡片 |
| 查 TDL | 问机器人「今天有什么」「励步这周还有什么」 | 返回任务清单卡片 |
| 更新 TDL | 发消息「斯坦星球那条改到周五」「标记已完成」 | 确认更新卡片 |
| 系统提醒你 | 自动 | 每日 08:30 推送到期/逾期，每周一推送周报 |
| 系统追你 | 自动 | 逾期 2 天推送追问卡片（带按钮） |
| 看周报 | 自动 | 每周一钉钉卡片 |
| 开通日历同步 | 首次收到系统提示后点一次链接 | 浏览器完成授权，后续 TDL 自动写入个人钉钉日历 |

## 三、卡片交互

系统不只发文字——发钉钉卡片，带按钮，操作不离开聊天窗口。

│ ├─ 负责人：@少巍                        │
│ ├─ 截止：2026-05-20 18:00              │
│ ├─ 业务线：斯坦星球 / 教务教学            │
│ ├─ 优先级：P1-重要不紧急                 │

### 上级简报卡片（🔮 推广期，MVP 不做）

推广期启用。管理者创建 TDL 后，系统每周向直属上级推送一张简约简报。上级提前知道团队在忙什么，不是等逾期升级才被动知道。

```
┌────────────────────────────────────────┐
│ 👤 @邢杨 本周重点（5.14-5.17）          │
│                                        │
│ 📌 进行中：4 条                         │
│ ├─ W38续费冲刺（截止 5.16）             │
│ ├─ 三公班级盘点（截止 5.18）            │
│ ├─ 新班rebuy跟盯（截止 5.15）           │
│ └─ 教师例会（每周）                     │
│                                        │
│ ⚠️ 本周到期：2 条                       │
│ 🔴 已逾期：1 条（Jacey理念冲突沟通）     │
│                                        │
│ [📋 查看全部] [💬 跟进]                 │
└────────────────────────────────────────┘
```

💡 和"逾期升级"的区别：逾期升级是"出事了通知你"，上级简报是"平时就让你知道团队在做什么"。推广期启用，MVP 自用期不需要。

## 五、推送文案基调

固定卡片文案只承担提示作用，基调保持真诚、克制、有鼓励性：

- 说事实，不指责
- 承认工作量，例如“今天到期，辛苦了”
- 追问优先表达支持，例如“是不是卡在什么地方了？”
- 功能按钮继续保持简短，不把情绪写进按钮里

## 六、异常场景处理

系统已在以下场景实现异常处理，未在交互设计阶段单独成文，现集中列出：

| 场景 | 触发条件 | 用户看到的效果 | 实现位置 |
|------|---------|--------------|---------|
| AI 提取失败 | AI 返回空或缺字段 | 退化为草稿卡片，缺字段标「待补充」 | `intake_service.py` |
| 非文本消息 | 用户发送语音、图片等当前不支持的消息 | 机器人明确提示“当前先支持文字录入”，不静默无响应 | `dingtalk_stream_bot.py` |
| 未开通日历授权 | 负责人还没授权个人日历 | TDL 正常创建，系统提示先点一次“开通我的日历同步” | `calendar_service.py` + `calendar_auth_service.py` |
| 日历同步失败 | 钉钉日历 API 不可用 | TDL 正常创建，审计记 `calendar_create_failed`，主流程不中断 | `calendar_service.py` → `sync_*_best_effort` |
| 草稿确认被拒 | 缺 `owner_id` 或 `due_at` | API 返回 409，卡片不消失，仍提示补字段 | `confirm_tdl` |
| 草稿超时未确认 | 草稿 24h 状态未变 | 系统自动取消（状态机已定义，触发器 V1.1 实装） | 02 §四 状态流转 |
| 卡按钮回调无法解析 | `action_id` 格式错误或非法 UUID | 返回 `handled=False`，不静默吞掉 | `parse_card_action_id` |
