# 04 · 交互方式

> 与 03 的关系：03 定义了每个功能模块做什么，04 定义用户怎么跟这些功能交互。
> 与 06 的关系：06 定义了 MVP 只做飞书交互，Web 仪表盘和语音交互推迟。

---

## 一、交互原则

**越轻越好。** 所有操作通过飞书机器人的对话和卡片完成，不需要装新 App、不打开网页。未来可以扩展到语音，但 MVP 就是飞书打字。

---

## 二、六种交互场景

| 你做什么 | 怎么触发 | 系统怎么回应 |
|---------|---------|------------|
| 创建 TDL | 飞书给机器人发消息 | 高置信度→返回「已创建」卡片；低置信度→返回确认卡片 |
| 查 TDL | 问机器人「今天有什么」「励步这周还有什么」 | 返回任务清单卡片 |
| 更新 TDL | 发消息「斯坦星球那条改到周五」「标记已完成」 | 确认更新卡片 |
| 系统提醒你 | 自动 | 每日 08:30 推送到期/逾期，每周一推送周报 |
| 系统追你 | 自动 | 逾期 2 天推送追问卡片（带按钮） |
| 看周报 | 自动 | 每周一飞书卡片 |

---

## 三、卡片交互

系统不只发文字——发飞书卡片，带按钮，操作不离开聊天窗口。

### 确认卡片（草稿模式）

```
┌────────────────────────────────────────┐
│ 📌 TDL 草稿                             │
│                                        │
│ 斯坦星球暑期班课程方案审核                │
│ ├─ 负责人：@少巍                        │
│ ├─ 截止：2026-05-20 18:00              │
│ ├─ 业务线：斯坦星球 / 教务教学            │
│ ├─ 优先级：P1-重要不紧急                 │
│ ├─ 等待：@教研组                        │
│ ├─ 完成标准：[待补充]                    │
│ └─ AI 置信度：85%                       │
│                                        │
│ [✅ 确认创建] [📅 改时间] [👤 改负责人]   │
│ [📝 补标准] [⚡ 改优先级] [🗑 忽略]      │
└────────────────────────────────────────┘
```

### 已创建卡片（自动模式）

```
┌────────────────────────────────────────┐
│ ✅ 已创建 TDL                           │
│                                        │
│ 励步暑期招生方案终稿                      │
│ ├─ 截止：今天 18:00                      │
│ ├─ P0-紧急重要                          │
│ └─ 下一步：集中精力完成，还有 6 小时        │
│                                        │
│ [✏️ 修改] [✅ 标记完成] [⏸ 暂缓]        │
└────────────────────────────────────────┘
```

### 追问卡片

```
┌────────────────────────────────────────┐
│ ⚠️ 励步招生方案终稿 已逾期 2 天           │
│                                        │
│ 需要关注一下吗？                         │
│                                        │
│ [✅ 已完成] [📅 延期] [🆘 需协助]        │
└────────────────────────────────────────┘
```

### 查询回复

```
用户：「帮我看看这周还有什么没做完」

机器人：
📋 本周剩余待办（5.14-5.17）
├─ 🔴 P0 今天到期
│   └─ 励步暑期招生方案终稿（今天 18:00）
├─ 🟡 P1 本周内
│   ├─ 斯坦星球课程大纲修订（周五）
│   └─ 飞书工资需求评审（周五）
└─ 💡 建议：今天集中搞定 P0，周五两条分开两天处理
```

### 上级简报卡片（推广期）

管理者创建 TDL 后，系统自动向他的直属上级推送一张简约简报——不是每条都推，而是每周汇总。上级可以在事情发生之前就看到下属本周/本月的重点，而不是等到逾期升级才被动知道。

```
┌────────────────────────────────────────┐
│ 👤 @邢杨 本周重点（5.14-5.17）          │
│                                        │
│ 📌 进行中：4 条                         │
│ ├─ W38续费冲刺（截止 5.16）             │
│ ├─ 三公班级盘点（截止 5.18）            │
│ ├─ 新班rebuy跟盯（截止 5.15）           │
│ └─ 教师例会（每周）                     │
│                                        │
│ ⚠️ 本周到期：2 条                       │
│ 🔴 已逾期：1 条（Jacey理念冲突沟通）     │
│                                        │
│ [📋 查看全部] [💬 跟进]                 │
└────────────────────────────────────────┘
```

💡 和"逾期升级"的区别：逾期升级是"出事了通知你"，上级简报是"平时就让你知道团队在做什么"。推广期启用，MVP 自用期不需要。

## 四、管理层级配置（推广期用）

⚠️ 待你确认：推广到管理层时，需要你提供一份名单。格式：

```
姓名 + 飞书 user_id + 直属上级 + 管辖业务线
```

示例：
```
少巍, shaowei, 无上级, [励步, 斯坦星球, 飞书工资]
张三, zhangsan, shaowei, [励步]
李四, lisi, shaowei, [斯坦星球]
```

这份名单决定了催收升级链的路由规则。MVP 自用期不需要，但可以先给我一个草稿看看结构。
