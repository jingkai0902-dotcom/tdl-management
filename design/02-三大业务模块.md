# 三大业务模块设计

## 一、催收模块

### 1.1 催收节奏

系统按固定周期自动触发催收动作：

| 周期 | 触发时机 | 催收内容 |
|------|---------|---------|
| 每日 | 每天 08:30 | 今日到期 + 昨日逾期未清的 TDL |
| 每周 | 每周一 08:30 | 本周到期汇总 + 上周完成率统计 |
| 每月 | 每月 1 日 09:00 | 本月关键里程碑 + 上月完成复盘数据 |
| 每季 | 季初第 1 个工作日 | 季度 OKR 对齐 TDL + 上季效果回收统计 |
| 每年 | 年初第 1 个工作日 | 年度战略 TDL 拆解 + 年度复盘 |

### 1.2 单条 TDL 催收规则

不是一次性提醒，是**渐进式升级**：

```
截止前 3 天 → 飞书消息提醒：「X 任务 3 天后到期」
截止当天 09:00 → 强提醒：「X 任务今天截止」
截止后 1 天 → 再次提醒 + 询问：「是否需要协助/延期？」
截止后 3 天 → 自动标识为「逾期」+ 同步推送给该管理者的上级
截止后 7 天 → 升级到少巍 + 自动生成催办摘要
```

### 1.3 催收升级链

基于管理层级关系动态路由升级：
- 你提供管理层名单 + 每个人的直属上级
- 系统根据 TDL 负责人自动匹配升级路径
- 当日 TDL 逾期 1 天 → 通知上级（按你说的「当天任务 1 天就通知 boss」）
- 跨周期 TDL（周/月级别）逾期按比例升级：周任务 3 天、月任务 7 天

### 1.4 TDL 自动抓取来源

| 飞书源 | 抓取逻辑 |
|--------|---------|
| 飞书日历 | 有「待办/TDL/任务」关键词的事件自动录入 |
| 飞书审批 | 审批发起后自动生成追踪 TDL（等审批结果） |
| 飞书消息 | @你 + 「待办/任务/做一下/处理」等意图识别 → 自动提取 |
| 飞书文档 | @你 的评论/任务分配自动提取 |
| 手动录入 | 飞书机器人对话直接创建 |

## 二、处理模块

这是系统的核心智能层。AI 拿到 TDL 后不是简单转发，而是**先处理再交付**。

### 2.1 AI 处理管线

每条 TDL 进入系统后，经历以下管线：

```
原始 TDL 输入
    ↓
[Step 1] 意图识别与标准化
    - 从自然语言/语音提取核心任务
    - 自动填充字段：业务线、职能域、优先级、流程阶段
    ↓
[Step 2] 智能拆解
    - 大任务自动拆成子任务链
    - 识别依赖关系（A 完成才能做 B）
    - 识别协作人（提到谁、需要谁参与）
    ↓
[Step 3] 节奏排序
    - 基于截止时间 + 优先级 + 依赖链 + 历史完成速度
    - 生成「今日优先处理」建议排序
    ↓
[Step 4] 管理方法论注入
    - 四象限分类（重要/紧急）
    - OKR 对齐检查（这个 TDL 关联哪个目标？）
    - 时间预估（基于历史同类任务耗时数据）
    ↓
[Step 5] 补充提醒生成
    - 检测遗漏：这个 TDL 没写截止时间？追问
    - 检测冲突：同一天 5 个 P0 全部到期？预警
    - 检测空档：某业务线连续 2 周没有 TDL？提醒
    ↓
[Step 6] 输出到用户
    - 飞书消息推送处理结果
    - TDL 入库，进入催收周期
```

### 2.2 处理输出示例

用户对飞书机器人说：
> 「下周三前要完成斯坦星球暑期班的课程方案审核，需要教研组配合」

AI 处理后返回：
```
已创建 TDL：
📌 斯坦星球暑期班课程方案审核
├─ 截止：2026-05-20（周三）18:00
├─ 业务线：斯坦星球 / 教务教学
├─ 优先级：P1-重要不紧急
├─ 协作：@教研组
├─ 子任务建议：
│   ├─ ① 收集各年级课程方案草案（5.15 前）
│   ├─ ② 教研组初审反馈（5.18 前）
│   └─ ③ 最终审核定稿（5.20 前）
├─ 关联目标：斯坦星球 Q3 产品升级
└─ ⚠️ 提示：5.20 当天你还有 2 个 P0 到期，建议提前到 5.19 完成
```

## 三、反馈模块

### 3.1 反馈输出的两个通道

| 通道 | 形式 | 内容 | 频率 |
|------|------|------|------|
| 飞书推送 | 结构化消息卡片 | 周报/月报摘要 + 关键预警 | 每周一 + 每月 1 日 |
| Web 仪表盘 | 可视化数据面板 | 长期趋势、行为分析、团队对比 | 实时可查 |

### 3.2 飞书推送内容

**周报模板：**
```
📊 本周 TDL 周报（5.11-5.17）
├─ 完成率：12/18（66.7%）↓ 较上周下降 8%
├─ 逾期任务：3 条（已自动升级）
├─ 按业务线：
│   ├─ 励步英语：4/5 完成 ✅
│   ├─ 斯坦星球：3/5 完成 ⚠️
│   └─ 飞书工资：5/8 完成 ❌（2 条逾期）
├─ 本周最高效日：周三（完成 5 条）
└─ ⚠️ 预警：飞书工资线连续 2 周完成率 < 60%
```

**月报模板：**
```
📈 5 月 TDL 月报
├─ 整体完成率：72%
├─ 平均响应时间：1.8 天（较上月改善 0.3 天）
├─ 行为洞察：
│   ├─ 你的高效时段：上午 9-11 点（完成 60% 的 TDL）
│   ├─ 拖延模式：P1 类任务平均 4 天才开始处理
│   └─ 瓶颈业务：飞书工资相关 TDL 逾期率最高（35%）
├─ 协作分析：
│   ├─ 最频繁协作者：@张三（14 条协作 TDL）
│   └─ 协作阻塞：涉及教研组的 TDL 平均完成周期最长
└─ 下月建议：
    ├─ 飞书工资线 TDL 建议委派或拆分
    └─ P1 任务设置更早的内部 deadline
```

### 3.3 Web 仪表盘

可视化面板包含：
- **TDL 健康度仪表盘**：完成率趋势、逾期率、响应时间曲线
- **行为热力图**：按日/周/时间段展示 TDL 完成密度
- **业务线对比雷达图**：各业务线完成率、逾期率、平均周期
- **协作网络图**：管理层之间的 TDL 协作关系可视化
- **个人效率档案**：长期累积的行为模式、精力曲线、改进轨迹

### 3.4 长期记忆沉淀

系统为每位管理者维护**行为档案**，跨周/月/年持续累积：
- 精力曲线模型（什么时段做什么类型任务最高效）
- 拖延模式库（什么类型任务容易被拖延，原因模式）
- 协作效率画像（跟谁协作顺畅，跟谁容易阻塞）
- 管理杠杆点（哪些 TDL 完成后的业务影响最大）

这些数据不随单次复盘消失，AI 会基于历史数据给出越来越精准的建议。
