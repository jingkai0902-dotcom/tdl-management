## 三、励步英语 组织关系

### 3.1 标准校区模型（理论）

```
                        校长
                         │
         ┌───────┬───────┼───────┬───────┐
         │       │       │       │       │
       市场     销售     教学   教学职能部  综合管控部
                              (教务+班主任) (人力+行政)
```

**五部门定位**：

| 部门 | 组成 | 定位 |
|------|------|------|
| 市场 | 线上+线下获客 | 铁三角之一，核心业务 |
| 销售 | 招生转化 | 铁三角之一，核心业务 |
| 教学 | 课程交付+教学品质 | 铁三角之一，核心业务 |
| 教学职能部 | 教务+班主任 | 与教学并列，独立第三方。教务管资源调配（排课/教具），班主任管家长服务和第三方检查反馈。不与教学合并以保持立场独立 |
| 综合管控部 | 人力+行政 | 区域化管理，不固定校区。行政主管由人力高潜人才选拔，设 HQ 直系领导向李珍汇报。统一采购、统一调配 |

💡 **设计考量**：
- 教务和班主任不与教学合并——防止教学经理将其工具化，失去第三方数据抽检作用
- 行政并入人力而非独立——行政发展空间有限，区域化管理加 HQ 领导层给行政主管提供上行通道
- 综合管控部区域化管理——人员跨校区调配，采购按品牌区域统一进行

**当前现实**：三个分校均无校长，铁三角中销售经理向时颖汇报而非向校长。

## 八、TDL 系统在此网络中的角色

这个组织关系网络在 TDL 系统里表现为三样东西：

1. **升级路由**：逾期通知沿汇报线向上流动
2. **协作通知**：当 TDL 的 `waiting_for` 或 `blocked_by` 触发时，系统知道"谁卡住了谁"，自动同步相关方
3. **上级简报**：每个汇报节点自动推送，不是李珍去盯，是系统催

这个文件独立于 TDL 系统——即使没有系统，它也是一份组织设计文档。有系统后，它变为自动化规则。
