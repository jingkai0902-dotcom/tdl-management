# 组织汇报路径搭建方案

> 目的：为李珍（CEO）与各部门/各品牌之间建立规范化、有节奏、有频率、有框架的信息回流渠道
> 优先级：教学部 > 前端运营（已成熟，锦上添花）> 财务/行政/人力（后续搭建）
> 状态：活文档，随业务调整持续修正

## 一、为什么需要这个文件

李珍作为企业大脑，获得的信息量直接决定策略判断和事务跟进的质量。当前最紧急的短板：**教学部信息回流不结构化**——会上提出的方向和任务，会后执行被"腰斩"，拖沓或变形，严重拖累教学成果呈现。

TDL 系统是工具，但汇报路径是制度。制度在先，工具辅助制度落地。这个文件独立于 TDL 系统，但 TDL 系统会承载它。

## 三、汇报路径设计原则

1. **不是"李珍盯人"，是"系统推动信息流动"**——信息应该在约定时间、以约定格式、自动到达李珍
2. **分层汇报**——日常运营问题在部门内解决；跨部门阻塞、偏离方向、需要决策的事项才向上流动
3. **证据驱动**——汇报内容必须有数据/截图/结果支撑，不能只凭嘴说
4. **品牌分开**——励步和斯坦各自有独立的汇报路径和节奏
5. **TDL 系统承载**——汇报路径的节点和输出物，由 TDL 系统自动提醒、追踪、汇总

## 四、教学部（励步）汇报路径——最高优先

### 当前问题诊断

- 李珍在会上提出方向和任务 → 会后落地被"腰斩"
- 教学团队执行力弱，"待做沟通"变成"一直没沟通"
- 张蕾处于教学终端但信息同步有断点
- 曹艳明/邢杨/韩雪敏/赵晓华的周度工作缺少向李珍的结构化输出

### 目标状态

李珍每周能收到一份来自教学部的**结构化周度简报**，包含：
- 本周完成的关键教学动作（数据支撑）
- 教师稳定性状态（红/黄/绿）
- 新师培训进度（姓名+阶段+下步）
- 续费数据（按校区+级别）
- 需要李珍决策或介入的事项（不超过 3 条）
- 上周李珍提出的方向和任务的落地状态（完成/进行中/未启动+原因）

### 汇报节奏

| 频率 | 内容 | 形式 | 负责人 |
|------|------|------|--------|
| 每周五 18:00 | 教学部周度简报 | 张蕾汇总 → 推送到李珍 | 张蕾 |
| 每两周一次 | 教学周会（张蕾主持，李珍旁听或参与） | 钉钉/飞书会议 | 张蕾 |
| 每月 | 月会述职 | 现有月会体系 | 曹艳明/邢杨/赵晓华/韩雪敏 |

### TDL 系统能帮什么

1. **每周五自动提醒张蕾**：「教学部周度简报截止」，同时向曹艳明/邢杨/赵晓华发送「提交本周教学数据给张蕾」
2. **李珍提出的方向自动转 TDL**：月会上说的事 → 系统生成 TDL → 分配到教学主管 → 每周简报里自动带上落地状态
3. **上级简报卡片**：每周向李珍推送张蕾汇总的教学简报（已在 04-交互方式 设计）

## 八、TDL 系统如何承载汇报路径

汇报路径的每个节点在 TDL 系统里表现为：

```
定频汇报节点（@recurring TDL）
  ├─ 负责人：汇报人
  ├─ 截止时间：固定频率（每周五/每月5号/每两周）
  ├─ 完成标准：简报必须包含的数据项
  ├─ 上级可见：李珍自动收到
  └─ 逾期升级：未提交 → 提醒本人 → 追问张蕾 → 升级李珍
```

这样汇报路径不是"李珍去盯"，而是系统自动执行。
